# Colisiones en tablas hash

## 1) Idea clave

Una tabla hash es rápida si el **factor de carga** es razonable. El problema inevitable son las **colisiones**.

Dos estrategias típicas:

- **Encadenamiento:** cada posición guarda una lista de elementos
- **Direccionamiento abierto:** buscas otra posición libre (sondeo)

## 2) Receta de examen

1. Define `m` (tamaño de la tabla) y función hash `h`.
2. Inserción:
   - encadenamiento: insertas en la cadena de `h(x)`
   - abierto: sondeo hasta encontrar hueco
3. Búsqueda: repites el mismo criterio (cadena o sondeo).

## 3) Errores típicos

- No tratar borrados en direccionamiento abierto (típicamente necesitas “tombstones”).
- Elegir `m` mal (depende del enunciado; muchas veces conviene primo).
